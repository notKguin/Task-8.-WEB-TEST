# Ручное тестирование (чек‑лист)

Дата: 2026-01-12

> Отметки можно ставить прямо в GitHub (☑ / ☐), либо вести как Issues с тегом `tested`.
> Ниже — базовый чек‑лист под требования задания.

## Функциональность

| Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|
| Нет «битых» ссылок | Пройтись по меню/кнопкам на главной, карточке события, профиле | Нет 404, все ссылки открываются | ☐ |
| Список событий | Открыть `/` | Список отображается, карточки кликабельны | ☐ |
| Детальная события | Открыть `/events/<id>/` | Отображаются данные события, кнопки «лайк/заявка» (если авторизован) | ☐ |
| Подача заявки | Авторизоваться → `/events/<id>/apply/` → отправить форму | Создаётся заявка, редирект на детальную, появляется сообщение | ☐ |
| Повторная заявка | Повторно отправить заявку на то же событие | Новая заявка не создаётся, показывается информационное сообщение | ☐ |
| Лайк | Нажать «лайк» (POST) | Лайк появляется/исчезает, редирект на детальную | ☐ |

## Формы и валидации

| Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|
| Регистрация: обязательные поля | `/signup/` → отправить пустую форму | Ошибки подсвечиваются, данные не сохраняются | ☐ |
| Регистрация: пароль/повтор | Разные `password1/password2` | Ошибка на `password2` | ☐ |
| Логин: неверные данные | `/login/` → неверный пароль | Ошибка, вход не выполняется | ☐ |
| Заявка: пустая мотивация | `/apply/` → пустое поле | Ошибка, сохранения нет | ☐ |

## Права доступа

| Роль | Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|---|
| Гость | Закрытые разделы | Открыть `/my/` и `/events/<id>/apply/` | Редирект на `/login/` | ☐ |
| Пользователь | Админ‑панель | Открыть `/admin/` | Нет доступа (редирект на admin login / запрет) | ☐ |
| Админ | Админ‑панель | Войти в `/admin/` | Доступ есть | ☐ |
| Админ | Экспорт XLSX | `/admin/export-xlsx/` → выбрать таблицы → Экспорт | Скачивается файл, корректные листы/колонки | ☐ |

## Надёжность и ошибки

| Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|
| Кастомная 404 | Открыть несуществующий URL | Показывается 404 без debug-страницы | ☐ |
| Некорректный ввод | Вводить «мусор» в формы | Ошибка валидируется, приложение не падает | ☐ |

## Производительность

| Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|
| Скорость загрузки | Открыть главную/детальную | < 2–3 сек (локально/в проде) | ☐ |
| N+1 на списке | Проследить запросы (Django Debug Toolbar/лог SQL) | Нет сотен запросов, используется `select_related/annotate` | ☐ |

## Безопасность

| Проверка | Шаги | Ожидаемый результат | Статус |
|---|---|---|---|
| DEBUG выключен | Проверить переменные окружения | В проде DEBUG=0 | ☐ |
| Хеширование пароля | Создать пользователя | В БД хранится хеш, не открытый пароль | ☐ |
| XSS | Попробовать сохранить `<script>` в поле (если возможно) | На странице отображается экранированно | ☐ |
| SQL‑инъекция | Попытка «инъекции» в логине/формах | Не проходит, нет ошибок/утечек | ☐ |
